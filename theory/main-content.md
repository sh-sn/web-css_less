# 📄 Основи HTML та CSS: Як працює сучасний веб

---

## 1. Вступ: Що таке веб-сторінка?

Будь-який сайт, який ви відкриваєте в браузері (Chrome, Safari, Firefox), — це набір текстових файлів, які браузер перетворює на візуальний інтерфейс.

Для створення цього інтерфейсу використовуються дві ключові технології:

- **HTML (HyperText Markup Language)** — мова розмітки гіпертексту. Це "скелет" або структура сторінки.
- **CSS (Cascading Style Sheets)** — каскадні таблиці стилів. Це "зовнішній вигляд" або дизайн сторінки.

> **Аналогія:** Уявіть будівництво будинку. **HTML** — це фундамент, стіни та дах. **CSS** — це колір шпалер, форма вікон та дизайн меблів.

### Як браузер читає файли?

Коли ви вводите адресу сайту, відбувається наступне:

1. Браузер відправляє **запит** на сервер.
2. Сервер повертає HTML-файл.
3. Браузер читає HTML, знаходить посилання на CSS та завантажує стилі.
4. Браузер будує **DOM (Document Object Model)** — дерево об'єктів сторінки.
5. Відбувається **рендеринг** — відображення фінального результату на екрані.

## 2. HTML: Структура та Теги

HTML складається з **тегів**. Тег — це команда для браузера, яка вказує, чим є конкретний фрагмент контенту (заголовком, абзацом чи картинкою).

### Базова структура HTML-документа

Кожен файл `.html` повинен мати таку структуру:

```html
<!DOCTYPE html>
<html lang="uk">
  <head>
    <meta charset="UTF-8" />
    <title>Мій перший сайт</title>
  </head>
  <body>
    <h1>Вітаю у світі веброзробки!</h1>
    <p>Це мій перший абзац тексту.</p>
  </body>
</html>
```

### Що означає кожна частина?

| Тег                      | Призначення                                                                         |
| ------------------------ | ----------------------------------------------------------------------------------- |
| `<!DOCTYPE html>`        | Оголошує версію HTML (HTML5). Обов'язковий перший рядок.                            |
| `<html lang="uk">`       | Кореневий елемент. Атрибут `lang` вказує мову для пошукових систем та скрінрідерів. |
| `<head>`                 | Службова секція. Не відображається на сторінці, але містить важливі метадані.       |
| `<meta charset="UTF-8">` | Кодування символів. Без нього кирилиця може відображатись як "крякозябри".          |
| `<title>`                | Заголовок вкладки браузера та результату у пошуку Google.                           |
| `<body>`                 | Весь видимий вміст сторінки знаходиться тут.                                        |

### Основні поняття

**1. Теги:** бувають парні та поодинокі.

- Парні: `<p>Текст абзацу</p>`, `<h1>Заголовок</h1>`
- Поодинокі (самозакриваючі): `<img />`, `<br />`, `<hr />`

**2. Атрибути:** надають додаткову інформацію тегу.

```html
<a href="https://google.com" target="_blank">Перейти в Google</a>
<img src="photo.jpg" alt="Моє фото" width="300" />
```

### Найпоширеніші HTML-теги

#### Заголовки

```html
<h1>Заголовок першого рівня (найбільший)</h1>
<h2>Заголовок другого рівня</h2>
<h3>Заголовок третього рівня</h3>
<!-- h4, h5, h6 — менші підзаголовки -->
```

> ⚠️ На сторінці має бути **лише один** тег `<h1>`. Це важливо для SEO (пошукової оптимізації).

#### Текст та форматування

```html
<p>Звичайний абзац тексту.</p>
<strong>Жирний текст</strong>
<em>Курсивний текст</em>
<br />
<!-- Перенос рядка -->
<hr />
<!-- Горизонтальна лінія-розділювач -->
```

#### Списки

```html
<!-- Маркований список -->
<ul>
  <li>Пункт 1</li>
  <li>Пункт 2</li>
  <li>Пункт 3</li>
</ul>

<!-- Нумерований список -->
<ol>
  <li>Перший крок</li>
  <li>Другий крок</li>
  <li>Третій крок</li>
</ol>
```

#### Зображення та посилання

```html
<!-- Зображення -->
<img src="images/cat.jpg" alt="Кіт на сонці" width="400" height="300" />

<!-- Посилання -->
<a href="https://example.com">Зовнішнє посилання</a>
<a href="about.html">Посилання на іншу сторінку сайту</a>
<a href="#section2">Якір — посилання на розділ цієї ж сторінки</a>
```

#### Таблиці

```html
<table>
  <thead>
    <tr>
      <th>Ім'я</th>
      <th>Вік</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Олена</td>
      <td>25</td>
    </tr>
    <tr>
      <td>Максим</td>
      <td>30</td>
    </tr>
  </tbody>
</table>
```

#### Форми

```html
<form action="/submit" method="POST">
  <label for="name">Ваше ім'я:</label>
  <input type="text" id="name" name="name" placeholder="Введіть ім'я" />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />

  <label for="message">Повідомлення:</label>
  <textarea id="message" name="message" rows="4"></textarea>

  <button type="submit">Надіслати</button>
</form>
```

## 3. CSS: Стилізація сторінки

Без CSS сайт виглядав би як чорно-білий текстовий документ. CSS дозволяє нам змінювати кольори, шрифти, розміри та розташування елементів.

### Синтаксис CSS

Правило CSS складається з **селектора** та **блоку оголошень**:

```css
селектор {
  властивість: значення;
}
```

**Приклад:**

```css
h1 {
  color: darkblue; /* Колір тексту */
  font-size: 32px; /* Розмір шрифту */
  text-align: center; /* Вирівнювання по центру */
}
```

### Як підключити CSS до HTML?

Існує три способи:

**1. Зовнішній файл (рекомендовано):**

```html
<link rel="stylesheet" href="styles/custom.css" />
```

**2. Тег `<style>` у `<head>` (для невеликих правил):**

```html
<head>
  <style>
    p {
      color: gray;
    }
  </style>
</head>
```

**3. Інлайн-стилі (не рекомендовано, важко підтримувати):**

```html
<p style="color: red; font-size: 18px;">Текст</p>
```

> 💡 Зовнішній файл — найкраща практика, оскільки дозволяє підключити один CSS до десятків HTML-сторінок одразу.

## 4. CSS-селектори

Селектор вказує, **до якого елементу** застосовувати стилі.

### Основні типи селекторів

```css
/* За тегом — всі абзаци на сторінці */
p {
  font-size: 16px;
}

/* За класом — елементи з class="highlight" */
.highlight {
  background-color: yellow;
}

/* За ідентифікатором — унікальний елемент з id="header" */
#header {
  background-color: navy;
}

/* Комбінований — тег <p> всередині <div> */
div p {
  color: darkgray;
}

/* Псевдоклас — стан при наведенні курсора */
a:hover {
  text-decoration: underline;
  color: orange;
}

/* Псевдоклас — перший дочірній елемент */
li:first-child {
  font-weight: bold;
}
```

### Клас vs. Ідентифікатор

|              | Клас (`.class`)                       | Ідентифікатор (`#id`)            |
| ------------ | ------------------------------------- | -------------------------------- |
| Символ       | `.` (крапка)                          | `#` (решітка)                    |
| Унікальність | Можна використати **багато разів**    | **Один раз** на сторінці         |
| Призначення  | Повторювані елементи (кнопки, картки) | Унікальні елементи (шапка, меню) |

```html
<!-- Клас — на кількох елементах -->
<p class="highlight">Текст 1</p>
<p class="highlight">Текст 2</p>

<!-- Ідентифікатор — лише на одному -->
<nav id="main-menu">...</nav>
```

## 5. Блокова модель (Box Model)

Це одна з найважливіших концепцій у CSS. Кожен елемент на сторінці браузер сприймає як прямокутну коробку.

```
+---------------------------+
|         MARGIN            |  ← зовнішній відступ (прозорий)
|  +---------------------+  |
|  |      BORDER          |  |  ← рамка
|  |  +---------------+  |  |
|  |  |    PADDING    |  |  |  ← внутрішній відступ
|  |  |  +---------+  |  |  |
|  |  |  | CONTENT |  |  |  |  ← сам контент (текст, зображення)
|  |  |  +---------+  |  |  |
|  |  +---------------+  |  |
|  +---------------------+  |
+---------------------------+
```

```css
div {
  width: 300px; /* Ширина контенту */
  height: 150px; /* Висота контенту */
  padding: 20px; /* Внутрішній відступ з усіх боків */
  border: 2px solid black; /* Рамка */
  margin: 30px; /* Зовнішній відступ */
}
```

### Box-sizing: важлива деталь

За замовчуванням `width` у CSS вказує ширину **тільки контенту**. Padding та border додаються зверху. Щоб спростити розрахунки, використовують `box-sizing: border-box`:

```css
/* Глобальне скидання — найкраща практика */
* {
  box-sizing: border-box;
}
```

Тепер `width: 300px` означатиме ширину **разом** з padding та border.

## 6. Позиціонування та Flexbox

### Потік документа

За замовчуванням елементи розміщуються у **нормальному потоці**: блокові елементи (`<div>`, `<p>`, `<h1>`) займають всю ширину і стоять один під одним, а рядкові (`<span>`, `<a>`, `<strong>`) — в один рядок.

### CSS Display

```css
div {
  display: block;
} /* блочний — займає весь рядок */
span {
  display: inline;
} /* рядковий — в потоці тексту */
img {
  display: inline-block;
} /* рядковий, але сприймає width/height */
.hide {
  display: none;
} /* приховує елемент повністю */
```

### Flexbox — гнучке розміщення

Flexbox — сучасний спосіб вибудовувати елементи в ряд або стовпчик.

```css
.container {
  display: flex;
  flex-direction: row; /* або column */
  justify-content: center; /* вирівнювання по горизонталі */
  align-items: center; /* вирівнювання по вертикалі */
  gap: 16px; /* відстань між дочірніми елементами */
}
```

```html
<div class="container">
  <div class="card">Картка 1</div>
  <div class="card">Картка 2</div>
  <div class="card">Картка 3</div>
</div>
```

**Значення `justify-content`:**

- `flex-start` — всі ліворуч
- `flex-end` — всі праворуч
- `center` — по центру
- `space-between` — рівномірно, перший та останній притиснуті до країв
- `space-around` — рівномірно з однаковими відступами

## 7. Кольори та Шрифти в CSS

### Способи задати колір

```css
p {
  color: red; /* Назва */
  color: #ff5733; /* HEX-код */
  color: rgb(255, 87, 51); /* RGB */
  color: rgba(255, 87, 51, 0.8); /* RGBA — з прозорістю */
  color: hsl(14, 100%, 60%); /* HSL — відтінок, насиченість, яскравість */
}
```

### Шрифти

```css
body {
  font-family: 'Arial', sans-serif; /* Сімейство шрифтів */
  font-size: 16px; /* Розмір */
  font-weight: bold; /* Насиченість: normal / bold / 400 / 700 */
  font-style: italic; /* Стиль: normal / italic */
  line-height: 1.6; /* Міжрядковий інтервал */
  letter-spacing: 0.5px; /* Відстань між символами */
}
```

### Підключення Google Fonts

```html
<head>
  <link
    href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap"
    rel="stylesheet"
  />
</head>
```

```css
body {
  font-family: 'Roboto', sans-serif;
}
```

## 8. Семантика — чому це важливо?

Сучасна веброзробка вимагає використання **семантичних тегів**. Вони допомагають пошуковим системам (Google) та людям з інвалідністю розуміти структуру сторінки.

### Семантична vs. Несемантична розмітка

**Погано (без семантики):**

```html
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="footer">...</div>
```

**Добре (з семантикою):**

```html
<header>...</header>
<nav>...</nav>
<main>
  <article>
    <section>...</section>
  </article>
  <aside>...</aside>
</main>
<footer>...</footer>
```

### Основні семантичні теги

| Тег            | Призначення                                |
| -------------- | ------------------------------------------ |
| `<header>`     | Шапка сайту або секції                     |
| `<nav>`        | Блок навігації (меню)                      |
| `<main>`       | Основний вміст сторінки (один на сторінці) |
| `<section>`    | Тематична секція сторінки                  |
| `<article>`    | Самостійний матеріал (стаття, пост)        |
| `<aside>`      | Бічна панель, додатковий контент           |
| `<footer>`     | Підвал сторінки                            |
| `<figure>`     | Ілюстрація з підписом                      |
| `<figcaption>` | Підпис до `<figure>`                       |
| `<time>`       | Дата або час                               |

## 9. CSS-змінні та Медіа-запити

### CSS-змінні (Custom Properties)

Дозволяють зберігати значення, що повторюються, в одному місці:

```css
:root {
  --primary-color: #3498db;
  --secondary-color: #2ecc71;
  --font-size-base: 16px;
  --border-radius: 8px;
}

button {
  background-color: var(--primary-color);
  border-radius: var(--border-radius);
  font-size: var(--font-size-base);
}
```

Якщо потрібно змінити основний колір — правимо лише `:root`, і зміна поширюється на весь сайт.

### Медіа-запити (адаптивний дизайн)

Медіа-запити дозволяють застосовувати різні стилі залежно від ширини екрану. Це основа **адаптивного (responsive) дизайну**.

```css
/* Стилі для мобільних (до 768px) */
@media (max-width: 768px) {
  .container {
    flex-direction: column; /* Переводимо у стовпчик */
  }

  h1 {
    font-size: 24px; /* Зменшуємо заголовок */
  }
}

/* Стилі для планшетів (768px — 1024px) */
@media (min-width: 768px) and (max-width: 1024px) {
  .container {
    gap: 12px;
  }
}
```

> 💡 Підхід **Mobile First**: спочатку пишемо стилі для мобільних, потім через `min-width` додаємо стилі для більших екранів. Це вважається найкращою практикою.

## 10. Практичний приклад: Проста картка

```html
<!DOCTYPE html>
<html lang="uk">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Картка продукту</title>
    <link rel="stylesheet" href="style.css" />
  </head>
  <body>
    <div class="card">
      <img class="card__image" src="laptop.jpg" alt="Ноутбук" />
      <div class="card__body">
        <h2 class="card__title">Ноутбук Pro X</h2>
        <p class="card__description">Потужний пристрій для роботи та навчання.</p>
        <span class="card__price">24 999 грн</span>
        <button class="card__button">Додати в кошик</button>
      </div>
    </div>
  </body>
</html>
```

```css
/* style.css */

* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;

body {
  font-family: 'Arial', sans-serif;
```

## Висновок

HTML та CSS — це мови, які не потребують складної логіки програмування (як Python чи JS) на старті, але вони є фундаментом веброзробки. Ключові теми, які ми розглянули:

1. **Структура HTML-документа** — теги, атрибути, семантика
2. **Підключення CSS** — зовнішній файл, `<style>`, інлайн
3. **Селектори** — за тегом, класом, ідентифікатором
4. **Блокова модель** — content, padding, border, margin
5. **Flexbox** — сучасне розміщення елементів
6. **Адаптивний дизайн** — медіа-запити та mobile-first підхід
7. **CSS-змінні** — для масштабованого коду

Розуміючи ці основи, ви зможете створити будь-який візуальний інтерфейс у майбутньому.
